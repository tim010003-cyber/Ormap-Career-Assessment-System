# 怎麼寫一篇文章

這個資料夾是文章的**唯一真實來源**。你在 Notion 或 Obsidian 寫好、匯出 Markdown，
放進 `content/posts/`，推上去之後網站就會自動長出排好版的文章頁、更新列表、
分類、RSS 與 sitemap。你不需要碰任何 HTML。

（批次 3 完成後，會有一個「貼上就發布」的後臺，連檔案都不用碰。）

---

## 檔案放哪、檔名怎麼取

```
content/posts/<英數檔名>.md
```

**檔名就是網址**。`content/posts/work-values-101.md` 會變成
`https://careervalue-ormap.web.app/blog/work-values-101.html`。

檔名規則：**只能用小寫英文、數字和連字號**（`a-z`、`0-9`、`-`）。
如果你想用中文檔名，就在 frontmatter 裡另外指定 `slug`。

> **文章發布之後不要改檔名／slug。** 改了等於換網址，舊連結會失效、
> 已經累積的搜尋排名也會歸零。真的需要改，跟我說，我會保留舊頁做轉址。

## 檔案長什麼樣

最上面用 `---` 包起來的區塊叫 frontmatter，是這篇文章的設定；下面就是正文。

```markdown
---
title: 為什麼「找熱情」常常讓人更焦慮
seo_title: 找不到熱情怎麼辦？三個更實際的起點｜原味藍圖
description: 「找到你的熱情」聽起來很美，但它常常讓人卡住。這篇談談為什麼，以及可以換成什麼問題。
category: career
tags: [工作價值, 職涯探索]
cover: /assets/blog/passion.jpg
date: 2026-07-25
author: 何昆陽
draft: false
---

正文從這裡開始，直接用 Markdown 寫就好。
```

### 欄位說明

| 欄位 | 必填 | 說明 |
|---|---|---|
| `title` | ✅ | 文章標題，會成為頁面大標與列表標題 |
| `seo_title` | | 搜尋結果顯示的標題。留白時自動用 `title｜原味藍圖` |
| `description` | ✅ | 一到兩句摘要。**同時用於** meta description、社群分享預覽、列表卡片，所以請認真寫 |
| `category` | ✅ | 只能填 `career`、`life`、`org` 其中一個，見下表 |
| `tags` | | 自由標籤，寫成 `[標籤一, 標籤二]` |
| `cover` | | 封面圖路徑。留白時用品牌預設圖 |
| `date` | ✅ | 發布日，格式必須是 `YYYY-MM-DD` |
| `updated` | | 更新日，同格式。有改動內容時建議補上 |
| `author` | | 作者名。留白時用「原味藍圖」 |
| `draft` | | 填 `true` 就**不會發布**，也不會進列表、RSS 與 sitemap。寫到一半就先放 `true` |

### 分類只有三種

| 填這個 | 對外顯示 | 放什麼 |
|---|---|---|
| `career` | 職涯與工作 | 工作價值、興趣、能力、履歷定位、職缺分析、轉換選擇 |
| `life` | 生涯與角色 | 人生角色、時間與精力配置、關係、生活設計、長期調整 |
| `org` | 組織與人才 | 職務設計、人才標準、甄選流程、主管判斷、組織合作 |

這三類和網站上「探索工具」的分類是同一套，不另外開第四類。

## 正文可以用什麼

標準 Markdown 都支援：標題、粗體、斜體、清單、引用、連結、圖片、表格、程式碼區塊。

**關於標題階層**：正文裡請從 `##` 開始（`#` 留給文章大標，系統會自動放）。
文章右側的目錄是從 `##` 和 `###` 自動抓的，所以段落標題下好，目錄就會漂亮。

**關於圖片**：Notion 裡的圖片直接複製過來只會是一個會失效的連結。
圖片要另外放進 `assets/blog/` 再用 `/assets/blog/檔名.jpg` 引用。
（批次 3 的後臺會提供拖拉上傳，自動處理這件事。）

## 本機預覽

```bash
npm run build:content
```

跑完會在 `blog/` 產生文章頁。用任何靜態伺服器打開來看即可。

如果有欄位填錯，它會直接告訴你哪個檔案、哪裡有問題，而且**不會產生任何檔案**。
在 CI 上也是一樣：檢查沒過就中止部署，線上網站維持前一版，不會被寫壞。

---

## 產生物不要手動編輯

以下檔案都是自動產生的，改了下次 build 就會被蓋掉，也不進版控：

- `blog.html`
- `blog/<slug>.html`
- `blog/category/<分類>.html`
- `feed.xml`
- `sitemap.xml`

要改文章，永遠是改 `content/posts/` 裡的 `.md`。
