## 1. 如何使用

- 本清單是第一輪靜態程式碼盤點，不宣稱已完成瀏覽器或正式環境驗證。
- 標示「已確認」表示目前程式碼直接引用；不代表設計良好，也不代表存取權限安全。
- 標示「待 Claude 確認」表示需要既有模組維護者核對事件順序、隱含相依或漏列掛點。
- 未確認前，Codex 可以進行 Index、Dashboard、文件與隔離的新頁面設計；不得對本清單涵蓋的評測／白板檔案進行跨結構重寫或全域 CSS 重構。
- 必須修改保護項目時，先提出舊介面、新介面、遷移方式、受影響頁面與回歸案例；不能直接刪除後要求另一模組配合。
- 本清單不授權修改 Firestore Rules、資料、正式環境或任何使用者紀錄。

## 9A.4 mirror／wb 生命週期（原§9 第 4 項要求）

**TWA mirror（`?mirror=1&sid=`）**：1164-1165 解析 → 1167-1170 清 localStorage → **2577 auth：未登入直接 return，不導轉、不訂閱**（2578-2581）→ 2594 `loadData()` → 2595 `init()`（1581 加 `body.mirror-mode`；**1226 `setTimeout(applyMirrorUI, 80)`**）→ 2598 `setupCatalogEvents()` → 2605-2614 `onSnapshot` → `applyMirrorLiveData` → **2614 return，跳過 `loadHistoryFromUrl`、`loadCloudDraft` 與 30 秒自動同步**。**無 cleanup**：unsubscribe 被丟棄。

**HTI mirror**：654-655 解析 → **658 清 localStorage（含 mirror 模式，見 9A.5 G3）** → 1137 `onload` → **1231 未登入直接導向 `../index.html`**（與 TWA 行為相反）→ 1267-1272 以 200ms **無上限輪詢**等 `activities` 載入 → 1274-1281 `onSnapshot`。**無 cleanup**。

**TWA wb（`?wb=1&sid=`）**：2618 設 `_wbSessionId` → 2622-2624 寫 `whiteboard_active/{uid}` → 2628-2642 進場同步（**此時 localStorage 已於 1167 清空，故必然寫入空值**）→ 之後由 `_wbDebouncedSync`(2553) 500ms debounce 觸發。**資料來源是 localStorage 而非記憶體**(2557)。

**HTI wb**：1239 設 `_wbSessionId` → 1246 banner → 1249 立即同步 → **1252 `setInterval` 每 3 秒無條件寫入**（OPS-002）→ 1255-1261 訂閱 `whiteboard_active/{uid}` 以偵測結束。

**已確認的順序性缺陷**：
- **B1（功能性，最明確）**：`whiteboard.html:510` 讀 `ld.scores`，但 `tools/hti.html:1158-1163` 寫的是 `finalScores`，**HTI 洞察列永遠顯示「尚未開始填答」**。正確欄位應為 `finalScores`（且僅 stage 3 才有值，前兩階段需另行呈現）。
- TWA 的 `liveData.insight` 與 `focusType` 有寫入但 `applyMirrorLiveData` 未讀取（2479-2516），鏡像端 `#final-insight` 永遠空白。
- `whiteboard.html:450` 以 `iframe.style.display === 'none'` 作為「只設一次 src」的旗標——**以 CSS 狀態當旗標**，任何改用 class／visibility 控制顯示的重構都會使其失效或永久卡死。
- `tools/twa.html:2467` 的 `main > section` 條件是「有 `.no-print` 且**沒有 id**」，但該檔所有 `main > section` 皆有 id → 該行實際上未隱藏任何元素，全靠 CSS 479 完成。

## 9A.5 允許 Codex 只改樣式的安全邊界（原§9 第 5 項要求）

**可自由調整**：色彩、字體、間距、陰影、圓角、非狀態性的 class、9A.3「僅視覺」列的全部項目、`brand.css` 的 token 定義。

**必須先提出遷移計畫**（不可只改樣式）：
1. **任何 `pointer-events` 相關規則。** mirror 唯讀**唯一**靠 `tools/twa.html:481` 與 `tools/hti.html:57` 的 `body.mirror-mode * { pointer-events:none !important }`。JS 端僅 `saveAll`(2408) 與 observer(1851) 有防護，slider `oninput`、checkbox `onchange` 皆無 `_isMirrorMode` 檢查。移除或被覆蓋即失去唯讀保證（且鍵盤操作本就不受此限——已登記為殘留風險）。
2. **`.stage.locked` 的 `pointer-events:none`**（`tools/twa.html:507-512`）：Gated-lite 的實際 gate。
3. **`.indicator-checkbox:checked + .ind-label`**：相鄰兄弟結構不可拆。
4. **`.quadrant-layout` 的顯式 grid 座標**與 `.quad-cell.expanded .quad-body` 父子選擇器。
5. **Tailwind `.hidden` 的狀態機用法**（全檔）。
6. **以 className 整串覆寫的元素**：`#pick-*`、`#status-area`、`#top-badge`——加在這些元素上的任何 class 都會被 JS 抹掉。
7. **`display` 被當作旗標讀取之處**：`whiteboard.html:450`。

**額外約束**：HTI 第一個 `<script>` 不得改為 module（見 9A.2）。
