## 5. 文件整理原則

- 不因內容相近就刪除或合併歷史文件。
- 不以修改日期直接判斷哪份文件勝出；依決策狀態與使用者最新指示判斷。
- 使用者校稿文件、產品基準、工程契約與歷史規格分開維護。
- 網站文案變更先進入可讀盤點，再映射到頁面／區塊 ID；程式碼不是使用者唯一的內容入口。
- 未經確認的內容標為「提案」「推論」或「尚未驗證」，不得寫成已定稿。
