## 1. 執行摘要

### 1.1 核心判斷

「工作價值觀羅盤」不只是價值觀分類測驗，而是一套職涯決策練習。它要求使用者完成以下轉換：

```text
模糊的不舒服
→ 辨識心理需求
→ 分開現況與理想
→ 在有限資源下取捨
→ 翻譯成可觀察的工作環境條件
→ 判斷保留、協商、查證或轉換的行動
```

因此，課程不應只介紹六大類與二十項需求。課堂最需要先教會的是：如何根據經驗評分、如何區分相近需求、如何做取捨、如何把抽象期待轉為可查證條件，以及如何避免把落差直接等同於離職理由。

### 1.2 建議的課程承諾

學員完成課程與線上工具後，應能：

1. 用心理需求而非好壞評價理解工作價值觀。
2. 分開描述「現在獲得多少」與「未來需要多少」。
3. 找出核心紅線、重要偏好與可調整空間。
4. 用具體事件與環境證據解釋自己的分數。
5. 把重要需求轉成面試、資訊訪談或職場協商問題。
6. 從職業倦怠中辨識可能的需求耗損來源。
7. 在離職前比較保留、協商、補充、職務調整與轉換等選項。
8. 把結果視為當下狀態的快照，而非固定人格或永久答案。

## 4. 從工具反推的課程知識骨架

## 4.1 單元一：價值觀是心理需求，不是人格好壞

### 必教知識

- 工作價值觀描述的是「環境提供什麼，我比較容易長期投入與感到滿足」。
- 它不等於能力、興趣、人格或道德優劣。
- 相同職業可以在不同組織中提供完全不同的需求滿足。
- 個人需求會隨生命階段、健康、家庭責任與職涯任務改變。

### 建議活動

提供三個職場案例，讓學員辨識同一個表面抱怨背後可能是不同需求。例如「主管不尊重我」可能指主管專業不足、專業意見未被採納、決策權不足或價值不契合。

### 需要避免

- 把利他分高解讀為比較善良。
- 把地位分高解讀為虛榮。
- 把舒適分高解讀為不努力。
- 把某次結果當成固定人格標籤。

## 4.2 單元二：辨識二十項需求的邊界

課堂不需要要求背誦所有定義，應集中處理容易混淆的對照：

| 容易混淆的需求 | 教學區分問題 |
|---|---|
| 發揮專長／發揮創意 | 我想用擅長的方法解題，還是想創造新的方法？ |
| 獨立作業／決策權力 | 我需要不被頻繁打擾，還是需要能影響資源與方向？ |
| 看見成果／獲得認可 | 我需要看見產出真正發生作用，還是需要貢獻被他人辨識與肯定？ |
| 社會貢獻／價值契合 | 我想創造正向影響，還是至少不做違反信念的事？ |
| 主管領導／主管專業 | 我在意主管如何對待與承擔，還是主管能否做出專業判斷？ |
| 公司穩健／制度合理 | 我擔心組織能否存續，還是規範、權責與對價是否公平？ |
| 責任擔當／決策權力 | 我想完整承擔成果，還是想擁有支配資源與方向的權力？ |

## 4.3 單元三：現況與理想的證據式評分

### 現況滿意度

要求學員回想目前或上一份工作，使用具體事件評分，不以單一情緒或單次衝突代表全部現況。

### 理想期待值

要求學員想像未來 1–3 年的工作階段，而不是無限制想像「完美公司」。

### 建議三點錨定

- 低：長期缺乏或不是目前優先；即使只有基本條件也可接受。
- 中：需要基本穩定存在，但可與其他條件交換。
- 高：長期缺乏會明顯耗損投入，選擇工作時必須主動查證。

### 教學提醒

- 分數是自我對話的索引，不是心理測量常模。
- 同分不代表兩項需求的具體條件相同。
- 0 分可以是有效答案，但要能說明「沒有也可以」的實際含義。

## 4.4 單元四：有限資源與有意識的取捨

將 115 分設計成「有限注意力與職涯資源」的模擬，而不是科學上的標準答案。

引導學員把結果分為：

1. 核心紅線：長期缺乏會造成顯著耗損或離開意圖。
2. 重要偏好：有會更好，可透過其他條件補償。
3. 彈性空間：目前階段可以接受較少。

建議追問：

- 如果只能保留三項，你保留什麼？
- 為了保留它，你願意放棄什麼？
- 這是你真實的內在需求，還是社會認為「好工作應該有」？
- 如果這項外顯條件被拿掉，你真正失去的是什麼感受？

## 4.5 單元五：從抽象需求到環境證據

學員需要練習四層轉譯：

```text
抽象需求
→ 可觀察條件
→ 可查證問題
→ 可採取行動
```

範例：

| 層次 | 內容 |
|---|---|
| 抽象需求 | 我需要專業被尊重 |
| 可觀察條件 | 專業建議被否決時，決策者會說明依據並允許討論 |
| 可查證問題 | 過去團隊意見與主管不同時，通常怎麼形成最後決策？ |
| 可採取行動 | 面試查證、資訊訪談、與主管協商決策規則 |

環境指標的 1／2 點權重應解釋為取得成本、稀少性及個人投入需求不同，不應稱為「低階／高階人格」。

## 4.6 單元六：從倦怠訊號回看需求耗損

課堂可引導學員把倦怠拆成可驗證假設：

- 是工作量與節奏問題，還是工作無法看見成果？
- 是主管互動耗損，還是主管專業不足造成反覆重工？
- 是薪資不足，還是付出與認可、權責不成比例？
- 是內容重複，還是沒有創意與決策空間？

需明示：工具可以協助辨識可能來源，但不能單獨診斷臨床倦怠，也不能取代醫療或心理專業評估。

## 4.7 單元七：結果解讀與離職安全護欄

### 必教原則

- 落差大是「需要進一步理解與行動」的訊號，不是立即離職指令。
- 現況高、理想高的項目，是轉換工作時容易忽略的既有資源。
- 現況高、理想低的盈餘，可以交換，但要先確認失去後是否有隱性影響。
- 理想期待反映目前階段，不是終身固定特質。
- 外顯條件應繼續追問其帶來的內在感受與意義。

### 離職前四步檢核

1. 證據：落差來自持續模式，還是單一事件？
2. 可調整性：能否透過溝通、職務重設、資源補充或邊界協商改善？
3. 替代環境：下一份工作真的更可能提供核心條件嗎？如何查證？
4. 轉換成本：財務、健康、家庭、時間與職涯風險是否可承受？

## 5. 建議課程流程

以下以 150–180 分鐘課程為提案；實際時間仍待依受眾與授課形式調整。

| 時間 | 單元 | 產出 |
|---:|---|---|
| 15 分 | 職場卡住的案例暖身 | 從表面抱怨找到可能需求 |
| 20 分 | 心理需求與環境適配 | 能區分價值觀、興趣、能力與人格 |
| 30 分 | 六大類與混淆需求辨識 | 完成需求對照練習 |
| 25 分 | 現況／理想評分校準 | 每人完成 2–3 項證據式試評 |
| 25 分 | 115 分取捨與紅線 | 找出紅線、偏好、彈性空間 |
| 30–35 分 | 環境指標與反向查證 | 寫出至少一題面試或協商問題 |
| 20–25 分 | 三態、九宮格與行動 | 完成離職前四步檢核 |
| 課後 20–30 分 | 線上完成全部評測 | 取得個人報告與反思紀錄 |

### 5.1 課堂與工具的責任分工

| 課堂優先負責 | 工具優先負責 |
|---|---|
| 建立概念邊界 | 逐項呈現需求定義 |
| 評分校準與具體事件練習 | 保存現況與理想分數 |
| 取捨的心理歷程 | 執行 115 分限制與指標點數 |
| 辨識外顯條件背後的內在需求 | 提供大量環境指標候選 |
| 避免衝動離職與固定化解讀 | 產生三態、九宮格、雷達與彙整 |
| 將結果轉成行動 | 保留報告供後續反思或諮詢 |

### 5.2 建議課後反思題

1. 最大落差帶來的是什麼情緒、身體感受或工作行為？
2. 哪個高分其實來自外界期待，而不是你當下真正想要的？
3. 哪個目前已經擁有的條件，換工作後最容易被你低估？
4. 哪一項可以先在目前工作中做小規模改善？
5. 如果考慮離開，你要如何查證下一個環境，而不是只相信職缺文案？

## 6. 已發現的校正事項

## 6.1 TWA-CONTENT-001：8 分的環境指標預算不一致

**確認事實**

- `tools/twa.html` 的說明文字寫「5 分 = 6 點、8 分 = 13 點」。
- 同檔 `BUDGET_MAP` 實際設定為 `5 → 6`、`8 → 12`。

**影響**

- 使用者可能誤以為系統少給 1 點。
- 課堂若引用 13 點，會與實際操作不一致。

**建議**

- 若目前演算法為 canonical，將文案改為「8 分 = 12 點」。
- 若設計意圖確實是 13 點，則需另行評估整張 `BUDGET_MAP`、既有資料與回歸影響，不應只改常數。

**建議優先度：P1 內容一致性。**

## 6.2 TWA-CONTENT-002：現況分數是否影響報告的說明矛盾

**確認事實**

- `tools/twa.html` 現況評分說明寫「這個分數不影響報告結論，只是讓你看見現實與理想的落差」。
- `report.html` 實際用現況與理想共同分類渴望／平衡／盈餘、九宮格、類別落差與最大落差。

**影響**

- 這不是次要參考值，而是報告的重要輸入。
- 使用者可能因誤以為不影響結果而較隨意作答。

**建議文案方向**

> 這個分數不代表你的能力或價值高低；系統會將它與理想期待相比，協助你看見目前的滿足、落差與可能的餘裕。

**建議優先度：P1 解讀正確性。**

## 6.3 TWA-CONTENT-003：「20 項環境指標」的數量與層級混淆

**確認事實**

- `data/twa-landing.json` 將工具描述為六大需求類別，「含 20 項環境指標」。
- 實際資料是六大類、20 項需求、281 個環境指標。

**影響**

- 使用者可能誤判作答規模。
- 「需求」與「環境指標」的概念層級不清楚。

**建議文案方向**

> 六大價值面向、20 項工作需求，並從 281 個具體環境指標中辨識你在意的條件。

若不希望公開 281 造成負擔，可寫：

> 六大價值面向、20 項工作需求，搭配數百個具體工作情境指標。

**建議優先度：P1 名詞與預期管理。**

## 6.4 TWA-STATE-004：0 分同時代表有效答案與未作答

**確認事實**

- 使用者已確認 0 分可以是真的不期待。
- 現有滑桿預設為 0，階段揭示以「分數 > 0」作為使用者已操作的主要訊號。
- `report.html` 在現況與理想皆為 0 時回傳未評分。
- 評測摘要以「理想分數 > 0」計算已評估需求數。

**影響**

- 「刻意選 0」與「尚未作答」無法可靠區分。
- 有效的 0／0 結果可能不出現在分類與完成統計中。
- 若直接改預設值或資料形狀，可能影響草稿、Firestore、報告、白板 mirror mode 與既有紀錄。

**建議處理順序**

1. 先做唯讀資料流盤點：確認是否已有 touched／answered 狀態可沿用。
2. 若沒有，提出最小狀態模型與舊資料相容策略。
3. 由使用者決定是否接受介面多一步確認，或採第一次操作即標記已作答。
4. 確認資料契約與回歸範圍後再實作。

**建議優先度：P1 產品語意／高回歸風險，需獨立工作包。**

## 6.5 TWA-WEIGHT-005：1／2 點權重的意義尚未完整對外說明

**確認事實**

- 工具目前以基礎／進階樣式呈現 1／2 點指標。
- 使用者確認權重意義包含成本較高、較稀少，以及個人需要投入更多熱情與需求。

**風險**

- 「進階」容易被理解成比較好、比較成熟或比較值得追求。
- 使用者可能為追求外顯理想形象，偏選昂貴條件而忽略內在感受。

**建議**

- 課堂與介面改用「一般條件／高投入條件」或其他不帶階級意味的命名提案。
- 文案明示：2 點表示通常更稀少、取得成本較高，並可能要求個人承擔更多投入；不代表一定比較適合。
- 正式更名前需確認目前名稱是否已被使用者定稿。
